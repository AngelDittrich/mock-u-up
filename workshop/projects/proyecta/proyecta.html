Proyecta UTCH

Student Portal UI/UX Refactor • Accessibility & Frontend Architecture

Proyecta UTCH is a frontend-focused refactoring project for the student portal of the Universidad Tecnológica de Chihuahua. The original system suffered from severe usability, accessibility, and visual consistency issues. This project redesigns the interface from a user-centered perspective, prioritizing clarity, accessibility, and maintainable frontend architecture while preserving the original system’s intent.

The Challenge

The legacy portal presented multiple critical problems: a fragmented visual hierarchy, broken layouts across screen sizes, low contrast text, and unintuitive navigation flows. Several UI components behaved inconsistently, and accessibility considerations were largely absent, making the system difficult to use for a significant portion of students.

Additionally, the frontend architecture was tightly coupled and hard to maintain. Styles were duplicated, component boundaries were unclear, and small visual changes often caused unexpected regressions elsewhere in the interface.

The challenge was to refactor the frontend without changing backend logic, delivering a modern, accessible, and visually coherent interface that could scale and be maintained long-term.

The Solution

The system was rebuilt using a mobile-first, component-based frontend approach. Semantic HTML was introduced to improve document structure and screen reader compatibility, while a global CSS architecture based on design tokens and custom properties ensures visual consistency across the platform.

Critical UI elements were decomposed into reusable components, each with its own scoped styles. This separation significantly improves maintainability and reduces unintended side effects when updating or extending the interface.

The redesign adheres to WCAG 2.1 Level AA accessibility standards, including keyboard navigation, visible focus states, proper color contrast, reduced-motion support, and explicit form labeling. The result is an interface that is not only visually improved, but functionally inclusive.

Design & UX Decisions

A clear visual hierarchy was established using scalable typography and spacing systems. Interactive elements were redesigned to provide immediate feedback and predictable behavior, reducing cognitive load for users.

The layout adapts fluidly across devices, ensuring that students can access services comfortably from mobile, tablet, or desktop environments. Touch targets, spacing, and navigation patterns were optimized for real-world usage rather than purely aesthetic considerations.

Impact & Results

The refactored portal delivers a significantly improved user experience, with better readability, navigation, and overall usability. The system now behaves consistently across browsers and screen sizes, reducing friction for daily student interactions.

From an engineering perspective, the new frontend architecture enables easier iteration, clearer separation of concerns, and long-term scalability. The project demonstrates the practical application of accessibility standards, modern CSS architecture, and user-centered design in an academic institutional context.
